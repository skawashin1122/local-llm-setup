## 応答時間の原因分析

### DeepSeek R1 が遅かった理由

DeepSeek R1は回答前に`<think>...</think>`タグで思考プロセスを大量生成：

```
プロンプト受信
  └→ <think>（2,000〜5,000 トークンの思考）</think>
  └→ 実際の回答（ユーザーが見たい部分）
```

- `stream: false` + トークン制限なしで全生成を待機したため441秒を記録
- Open WebUI でのストリーミング表示では実用可能だが、他モデルと比較して優位性なし

### mistral/gemma2/phi3 が遅かった理由

- **アーキテクチャの世代差**: 2023年以前の旧設計
- **量子化の非効率**: VRAM使用パターンが最適化されていない
- **Blackwell GPU最適化不足**: NVIDIA RTX PRO 2000への最適化が不十分


## 速度差の原因

### なぜqwen2.5:7bが速いのか？
1. **2024年の最新アーキテクチャ**: Qwen 2.5シリーズは推論効率に特化
2. **KVキャッシュ最適化**: メモリアクセスパターンが効率的
3. **量子化の質**: 高品質な4-bit量子化でVRAM使用を抑制

### なぜllama3.2:3bが速いのか？
1. **小型モデル**: 2GBでVRAMに完全に収まる
2. **最新のLlama 3.2**: 推論速度に最適化された設計
3. **効率的なアテンション機構**: 計算コストが低い

### なぜ他のモデルが遅いのか？
- **mistral:7b**: 2023年の古いアーキテクチャ、最適化が遅れている
- **gemma2:9b**: 5.4GBでVRAM制約に引っかかる、スワップ発生
- **phi3:3.8b**: Microsoft製だがBlackwellアーキテクチャ最適化が不十分
